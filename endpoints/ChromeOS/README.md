## Overview

ChromeOS device management for organisations operating in Microsoft-centric
environments. Covers Google Admin Console management of ChromeOS devices
alongside conditional access integration with Microsoft Entra ID, enabling
ChromeOS endpoints to access Microsoft 365 resources under policy-enforced
compliance requirements.

---

## Planned Content

The following topics are planned for this section:

- **ChromeOS Device Enrolment**: Enterprise enrolment via Google Admin Console,
  zero-touch provisioning, and domain join workflows
- **Google Admin Console**: Organisational unit structure, policy inheritance,
  and device policy management for ChromeOS fleets
- **Microsoft 365 on ChromeOS**: Configuration of Microsoft 365 web apps,
  Progressive Web Apps (PWAs), and Android app delivery via managed Google Play
- **Conditional Access Integration**: Entra ID conditional access policies
  targeting ChromeOS device compliance, certificate-based access, and
  device posture evaluation
- **ChromeOS Compliance and Device Posture**: Verified Access, device
  attestation, and integration with Microsoft compliance frameworks
- **Entra ID SSO for ChromeOS**: Single sign-on configuration for Microsoft
  resources from ChromeOS endpoints
- **Google Admin Console and Intune Co-management Patterns**: Hybrid management
  approaches for organisations with both Intune and Google Workspace
